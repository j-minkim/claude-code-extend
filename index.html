마켓플레이스가 곧 앱스토어라는 비유로 들어가세요. 정체는 플러그인 목록을 담은 git 저장소/폴더(.claude-plugin/marketplace.json)이고, 추가는 /plugin marketplace add owner/repo입니다. 공식 마켓 둘을 기억시키세요 — claude-plugins-official(앤트로픽 큐레이션, 첫 실행 시 자동 등록)과 claude-community(커뮤니티, anthropics/claude-plugins-community로 추가 → @claude-community로 설치). 서드파티도 owner/repo로 추가되며, 오늘 쓸 OMC가 그 예(Yeachan-Heo/oh-my-claudecode)입니다.

오해 방지 · 정확한 구조

지난주 메뉴 룰렛에 DB를 연결

[메뉴 룰렛 앱] ──(앱 코드)──▶ [menus.db] ◀──(MCP)── [Claude] 화면에 메뉴 표시 같은 SQLite "치킨 넣어줘"
✅ 정확한 이해
Claude가 MCP로 DB를 고치고 → 앱이 그 DB를 읽으니 → 화면에 나타남.
❌ 오해
"MCP가 앱에 데이터를 쏴준다" — 아니에요. MCP는 Claude↔DB 개발용 다리.
오해를 미리 막는 구조 설명 슬라이드예요. 다이어그램으로 [앱]→[menus.db]←[Claude(MCP)] 관계를 보여주세요. 정확한 이해는 "Claude가 MCP로 DB를 고치고 → 앱이 그 DB를 읽으니 화면에 나타난다"입니다. 흔한 오해 "MCP가 앱에 데이터를 쏴준다"는 명확히 부정하세요. 이 구조를 잡아야 다음 DB 연결·실시간 연동 실습이 이해됩니다.
락스텝 · 메뉴 룰렛 DB 연결

앱 띄우고 mcp-sqlite 연결

  • 1
    통합 터미널에서 앱 폴더: node server.mjs → localhost:4555 (메뉴 6개 확인)
  • 2
    .mcp.json에 sqlite 추가(경로 = 앱의 menus.db 절대경로) → 재시작
{ "mcpServers": { "sqlite": { "command": "npx", "args": ["-y", "mcp-sqlite", "/절대경로/menus.db"] } }} # 또는 터미널: claude mcp add sqlite -- npx -y mcp-sqlite /절대경로/menus.db # 확인: claude mcp list → sqlite ... ✓ Connected
메뉴 룰렛 DB 연결 락스텝이에요. 먼저 앱을 띄워(node server.mjs → localhost:4555) 메뉴 6개를 확인하고, .mcp.json에 sqlite를 추가하되 경로는 반드시 앱의 menus.db 절대경로를 쓰세요. 여기서 막히면 거의 경로 문제입니다 — claude mcp list로 ✓ Connected를 확인하고 넘어가세요. 안 되면 트러블슈팅 표를 펼치면 됩니다.

🧩 "실시간"의 정확한 뜻 — Claude가 고치면 DB는 즉시 바뀌고, 화면은 새로고침(다음 fetch) 때 반영. 진짜 푸시(소켓)는 추가 작업.
오늘의 하이라이트 슬라이드예요. 쓰기 전에 예측 박스로 "새로고침하면 어떻게 될까?"를 먼저 물어 학생 참여를 끌어내세요. 읽기 → 쓰기 → 집계 → 수정 순서로 시연하고, 국밥·돈카츠·쌀국수 3개가 새로고침에 등장하는 순간이 가장 임팩트 있습니다. "실시간"은 푸시가 아니라 다음 fetch 때 반영이라는 점을 정확히 짚고, 다음 슬라이드(로컬 vs 클라우드)로 자연스럽게 넘어가세요.
언제 뭘 쓰나

파일 DB로 감 잡았으면, 이제 클라우드 DB

로컬 SQLite (mcp-sqlite)
파일 1개가 곧 DB. 설치·계정 0. 학습·오프라인·혼자 쓸 때. (← 방금 우리가 한 것)
Supabase (클라우드 Postgres)
계정·프로젝트가 필요. 대신 실제 웹앱·배포·여러 사용자가 동시에 같은 DB를 쓸 수 있어요.
🧩 여러분이 만들 웹앱은 보통 클라우드 DB를 씁니다 — 그 대표가 Supabase(관리형 Postgres)예요. 파일 DB는 혼자 연습용, 클라우드 DB는 진짜 서비스용. 그리고 다음 장에서, 자연어로 명령만 해서 클라우드 DB를 다뤄봅니다(SQL은 Claude가 대신 써요 · 전원 실습).
SQLite로 "DB를 MCP로 만진다"는 감을 잡았으니, 이제 진짜 서비스가 쓰는 클라우드 DB로 확장하는 다리예요. 핵심 대비는 하나입니다 — 로컬 SQLite는 파일 1개·계정 0·혼자 연습용, Supabase는 계정·프로젝트가 필요한 대신 실제 웹앱·여러 사용자가 동시에 쓰는 클라우드 Postgres. "여러분이 배포할 앱은 보통 클라우드 DB를 쓴다, 그 대표가 Supabase"라는 한 문장만 남기면 됩니다.
개념 + 연결법

앱은 SDK로 읽고, Claude는 MCP로 관리

웹앱은 supabase-js(클라이언트 SDK)로 DB를 직접 읽고 씁니다. Claude는 MCP로 같은 DB를 관리(테이블 만들기·SQL·행 추가)하고요.

오해 방지 — MCP는 Claude ↔ Supabase 관리용 다리예요. 앱에 데이터를 직접 쏘는 게 아니라, 같은 DB를 Claude가 만지면 앱이 그걸 읽습니다. (SQLite 데모와 같은 논리)

{ "mcpServers": { "supabase": { "type": "http", "url": "https://mcp.supabase.com/mcp?project_ref=${SUPABASE_PROJECT_REF}", "headers": { "Authorization": "Bearer ${SUPABASE_ACCESS_TOKEN}" } } }} # 안전: ...?project_ref=abc123&read_only=true (한 프로젝트만, 읽기전용)
인증 (둘 중 하나)
기본은 OAuth — 서버 URL만 등록하면 브라우저 로그인으로 인증. 대안은 PAT(개인 액세스 토큰) — 발급: supabase.com/dashboard/account/tokens → 위처럼 Authorization: Bearer 헤더로. 키는 환경변수(${SUPABASE_ACCESS_TOKEN})로 참조해 노출 방지.
안전 팁
read_only=true(읽기전용 Postgres — 탐색 시 안전), project_ref=<프로젝트id>(한 프로젝트로 한정, 계정 관리도구 비활성). 제공 도구(개략): list_tables·execute_sql·apply_migration·get_project_url·generate_typescript_types 등 ~30종.
여기서 학생이 가장 헷갈리는 지점을 먼저 끊고 가세요 — MCP는 앱에 데이터를 쏘는 통로가 아니라 Claude가 같은 DB를 관리하는 다리이고, 앱은 supabase-js로 그 DB를 직접 읽습니다(SQLite 데모와 완전히 같은 논리). 연결은 호스티드 HTTP MCP라 .mcp.json에 type:http + url + (PAT면)Authorization 헤더만 넣으면 됩니다. 인증은 기본 OAuth(URL만 등록→브라우저 로그인), 대안 PAT(대시보드 account/tokens에서 발급). 탐색은 read_only=true, 범위는 project_ref로 한정하라고 강조하세요. ⚠️ 당일 재검증: MCP URL/파라미터·대시보드 경로는 변동 가능하니 라이브 전 supabase 공식 문서로 한 번 더 확인.
락스텝 · 전원 다 같이 — LLM으로 쿼리 연습

  • 1

  • 2
    .mcp.json에 supabase 블록 → OAuth 로그인(URL만 등록·브라우저 인증, 토큰 복사 불필요) → 세션 재시작
  • 3

  • 4

  • 5

SQL은 직접 안 써도 돼요. 위처럼 "이렇게 해줘"라고 자연어로 명령만 하면 Claude가 알아서 테이블·쿼리를 만들어 실행합니다. (만든 SQL은 결과 옆에 참고로 보일 뿐, 외울 필요 없어요.) 전원 실습 — OAuth면 연결 30초, 각자 본인 프로젝트라 자유롭게.

Supabase 무료 티어 제약(연습엔 충분) — 무료 프로젝트 2개까지 · DB 500MB · 파일 1GB · 트래픽 5GB/월 · 자동 백업 없음. ★1주 동안 안 쓰면 프로젝트가 자동 일시중지(pause) → 대시보드에서 다시 켜면(restore) 돼요.

★전원 락스텝(시연 아님). 학생이 SQL을 직접 쓰는 게 아니라, "이렇게 명령하면 된다"를 보여주는 게 목표예요 — 자연어 명령 예시(테이블 만들기 → 넣기 → active만 → 개수 세기)를 차례로 시키고, Claude가 만든 SQL은 "이런 게 만들어졌네" 정도로 가볍게 짚으면 됩니다(외우게 하지 말 것). 연결은 OAuth 30초(토큰 복사 없음) — 계정·프로젝트만 미리 만들어 오게 하면 매끄럽습니다. 결과는 Supabase 대시보드 Table/SQL Editor로 교차 확인. ⚠️무료 티어: 프로젝트 2개·DB 500MB·트래픽 5GB·백업 없음, 1주 미사용 시 자동 pause(restore 가능) — 강의 후 안 써서 멈춰도 정상이라고 미리 안내. ⚠️ 당일 재검증: MCP URL/파라미터·대시보드 경로·무료 티어 수치(supabase.com/pricing).

API 키 + MCP를 합친 실제 예시예요. GitHub PAT를 발급해 MCP에 연결하면 "내 repo의 열린 이슈 정리"를 Claude가 해줍니다. 이 단계는 조금 어려우니, 앞의 개념·발급 원리만 확실히 잡고 GitHub 연결은 강사 시연으로 봐도 충분하다고 부담을 덜어주세요. .mcp.json이면 토큰 대신 ${GITHUB_PAT} 환경변수 참조를 권합니다.
보강 · 코드로 직접 (전원 락스텝)

같은 키를, 코드로도 직접 써봐요

🧩 API 호출 = 정해진 주소(URL)로 HTTP 요청 → JSON 응답을 받는 것. API 키는 Authorization 헤더에 넣는 출입증. (설치 0 — Node 22 내장 fetch)
// 1) 키 없이 — 공개 정보 (누구나) fetch("https://api.github.com/users/octocat") // 2) 키 넣고 — '나'로 인증 (딱 한 줄 추가!) fetch("https://api.github.com/user", { headers: { Authorization: "Bearer " + 토큰 } // ← 익명 → 나 })
락스텝 (실습 자료/API 코드 실습/)
node 1-공개호출.mjs(공개) → cp .env.example .env에 토큰 → node --env-file-if-exists=.env 2-내계정호출.mjs(내 계정·내 레포).
MCP vs 코드
MCP=Claude가 호출(채팅) · 코드=내 앱이 호출(스크립트). 같은 API·같은 키·같은 원리.

직접 안 짜도 OK — "내 열린 이슈도 보여줘", "별 많은 순 top3"처럼 Claude에게 코드 수정을 시키고 다시 실행하면 됩니다.

MCP로 붙였던 그 GitHub PAT를 이번엔 코드에서 직접 씁니다. 핵심 메시지는 딱 하나 — 1단계(키 없이 공개)와 2단계(키 넣고 나) 코드가 거의 같고, Authorization: Bearer 토큰 한 줄이 "익명 → 나"로 바꿔준다. 그게 API 키의 정체예요. 설치는 0(Node 22 내장 fetch + --env-file-if-exists). 학생은 코드를 외울 필요 없이 Claude에게 "이렇게 바꿔줘"로 변형하면 됩니다. 키는 .env에 두고 git 금지·노출 시 재발급을 한 번 더 강조. 자세한 절차는 실습 자료/API 코드 실습/README.md.
